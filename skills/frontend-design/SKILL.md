# Frontend Design

## Overview

Create distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements for a component, page, application, or interface. They may also provide context about audience, purpose, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:

- Define the purpose. Identify what problem the interface solves and who uses it.
- Choose an extreme tone. Consider directions such as brutally minimal, maximalist chaos, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, pastel, or industrial. Pick one clear direction and make it feel intentional.
- Capture the constraints. Respect framework, performance, accessibility, and product requirements.
- Decide the differentiator. Identify the one thing a user should remember after seeing the interface.

Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism can both succeed; intentionality matters more than intensity.

Then implement working code that is:

- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear aesthetic point of view
- Meticulously refined in the details

## Frontend Aesthetics Guidelines

Focus on:

- Typography: Choose fonts with character and purpose. Avoid generic defaults such as Arial, Inter, Roboto, and system stacks when the project allows alternatives. Pair a distinctive display face with a refined body face.
- Color and theme: Commit to a cohesive visual system. Use CSS variables or design tokens for consistency. Prefer dominant colors with deliberate accents over timid, evenly distributed palettes.
- Motion: Use animation to create high-impact moments and polished micro-interactions. Prefer CSS-only solutions for simple HTML work. Use Motion for React when available. Favor a few well-orchestrated reveals over many low-value effects.
- Spatial composition: Use asymmetry, overlap, diagonal flow, grid-breaking moments, generous negative space, or controlled density when they support the concept.
- Backgrounds and details: Build atmosphere instead of defaulting to flat fills. Use textures, gradient meshes, patterns, layered transparencies, dramatic shadows, decorative borders, grain, or custom cursors when they support the aesthetic.

Avoid generic AI-generated aesthetics such as predictable component patterns, overused font choices, clichéd color schemes, and interchangeable layouts. Vary light and dark themes, typography, and composition choices across tasks instead of converging on the same visual recipe.

If the work extends an existing product or design system, preserve the established brand language, spacing logic, and interaction patterns. Push for distinctiveness through composition, hierarchy, motion, and detail rather than replacing the system wholesale.

Match implementation complexity to the aesthetic vision. Maximalist designs require richer code, layered effects, and denser interactions. Minimalist or refined designs require restraint, precision, and exceptional control of spacing, typography, and subtle detail.
